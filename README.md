# ConfigCollectorWindows (Fork Edition)

این پروژه یک ابزار دسکتاپ ویندوزی (Rust + egui) برای جمع‌آوری کانفیگ از کانال‌های عمومی تلگرام است.

## تغییرات این فورک
- رابط کاربری فقط **انگلیسی** است و گزینه انتخاب زبان حذف شده است.
- توضیحات همه گزینه‌های برنامه در همین README به فارسی نوشته شده‌اند.
- آدرس‌های وابسته به ریپوی قبلی در کد به فورک فعلی تغییر داده شده‌اند.

## پیش‌نیازها
1. Rust را نصب کنید: https://www.rust-lang.org/tools/install
2. در ویندوز بهتر است Visual Studio Build Tools هم نصب باشد.

## اجرا
```bash
cargo build --release
.\target\release\config-collector-windows.exe
```

## توضیح فارسی گزینه‌های UI
### Main Settings
- **Check interval (minutes)**: هر چند دقیقه یک بار جمع‌آوری تکرار شود.
- **Max pages per channel**: حداکثر تعداد صفحه‌ای که از هر کانال تلگرام خوانده می‌شود.
- **Look back days**: فقط پیام‌های این تعداد روز اخیر بررسی می‌شوند.

### Proxy
- **Type**:
  - `No proxy`: غیرفعال‌کردن کامل پروکسی (حتی پروکسی سیستم)
  - `System proxy`: استفاده از پروکسی سیستم‌عامل (قابل فعال/غیرفعال از همین لیست)
  - `HTTP`: استفاده از پروکسی HTTP دستی
  - `SOCKS5`: استفاده از پروکسی SOCKS5 دستی
- **Proxy host / Port / Username / Password**: فقط وقتی `HTTP` یا `SOCKS5` انتخاب شود استفاده می‌شوند.


### Output modes
- **Replace with new-only output (output/new_only)**:
  - فقط کانفیگ‌های جدید هر دور را ذخیره می‌کند.
  - محتوای قبلی همان فایل‌ها با خروجی جدید جایگزین می‌شود.
- **Append unique output (output/append_unique)**:
  - کانفیگ‌های جدیدِ غیرتکراری را به خروجی‌های قبلی اضافه می‌کند.
  - خروجی تجمعی نگه می‌دارد و موارد تکراری را دوباره اضافه نمی‌کند.
- هر دو حالت مستقل‌اند و کاربر می‌تواند هر کدام را جداگانه فعال/غیرفعال کند.

### Protocols (Enable/Disable + Limit)
- لیست پروتکل‌ها نمایش داده می‌شود.
- تیک هر پروتکل:
  - روشن = فعال برای خروجی
  - خاموش = حذف از خروجی
- **Max** روبه‌روی هر پروتکل: سقف تعداد خروجی همان پروتکل در هر نوبت.

### دکمه‌ها
- **💾 Save channels & settings**: ذخیره کانال‌ها و تنظیمات در فایل.
- **▶️ Start**: شروع فرآیند جمع‌آوری.
- **⏹ Stop**: توقف فرآیند.

### بخش خروجی و لاگ
- **Total new**: تعداد کانفیگ جدید پیدا شده در آخرین نوبت.
- شمارنده‌های پایین آن: تعداد هر پروتکل.
- **Channels list (one per line)**: هر خط یک کانال، مثل:
  - `@channel_name`
  - `https://t.me/channel_name`
- **Professional logs**: لاگ لحظه‌ای اجرای برنامه.

## فایل‌های خروجی
- `output/new_only/<protocol>.txt` و `output/new_only/mixed.txt`: خروجی حالت جایگزینی (فقط جدیدها)
- `output/append_unique/<protocol>.txt` و `output/append_unique/mixed.txt`: خروجی حالت تجمعیِ غیرتکراری
- `output/sent_history.json`: تاریخچه کانفیگ‌های ارسال‌شده برای جلوگیری از تکرار
