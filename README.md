# ConfigCollectorWindows

این پروژه یک نرم‌افزار دسکتاپ ویندوزی (Rust + egui) است که از کانال‌های عمومی تلگرام کانفیگ‌ها را جمع‌آوری می‌کند، بر اساس پروتکل جدا می‌کند، خروجی `mixed` می‌سازد و فقط کانفیگ‌های **جدید** را در هر دور خروجی می‌دهد.

## قابلیت‌ها
- رابط کاربری کامل فارسی
- اجرا به‌صورت لوکال و سبک (مناسب سیستم‌های ضعیف)
- وارد کردن لیست کانال‌ها در UI (هر خط یک کانال) + ذخیره
- زمان‌بندی خودکار (پیش‌فرض هر ۵ دقیقه)
- تنظیم بازه زمانی بررسی پیام‌ها (پیش‌فرض ۱ روز)
- فعال/غیرفعال کردن هر پروتکل + تعیین سقف تعداد خروجی هر پروتکل
- پشتیبانی پروکسی `HTTP` و `SOCKS5`
- تاریخچه ارسال برای جلوگیری از خروجی تکراری (`output/sent_history.json`)
- پاک‌سازی خودکار تاریخچه بر اساس تعداد روز تنظیم‌شده تا فایل‌ها سنگین نشوند
- لاگ حرفه‌ای داخل برنامه + `logs/app.log`

## پیش‌نیازها
1. نصب Rust (پیشنهادی از سایت رسمی):
   - https://www.rust-lang.org/tools/install
2. روی ویندوز بهتر است **Visual Studio Build Tools** هم نصب باشد.

## اجرای برنامه (توسعه/محلی)
در ریشه پروژه:

```bash
cargo build --release
.\target\release\config-collector-windows.exe
```

## راهنمای استفاده نرم‌افزار
1. برنامه را اجرا کنید.
2. در بخش **لیست کانال‌ها**، هر خط یک کانال وارد کنید. مثال:
   - `@channel_name`
   - `https://t.me/channel_name`
3. در بخش تنظیمات:
   - بازه چک خودکار را مشخص کنید (دقیقه).
   - تعداد روز بررسی پیام‌های قدیمی را مشخص کنید.
   - حداکثر صفحات قابل بررسی برای هر کانال را تنظیم کنید.
4. در بخش پروتکل‌ها:
   - هر پروتکل را فعال/غیرفعال کنید.
   - سقف تعداد خروجی همان پروتکل را تعیین کنید.
5. اگر نیاز داشتید، پروکسی HTTP/SOCKS5 را تنظیم کنید.
6. روی **ذخیره کانال‌ها و تنظیمات** بزنید.
7. روی **شروع** بزنید.
8. لاگ‌ها را در UI ببینید یا فایل `logs/app.log` را بررسی کنید.

## ساختار خروجی
- `output/<protocol>.txt` برای هر پروتکل فعال
- `output/mixed.txt` خروجی ترکیبی
- `output/sent_history.json` تاریخچه کانفیگ‌های ارسال‌شده

> تاریخچه خروجی بر اساس `lookback_days` پاک‌سازی می‌شود تا فایل‌ها بعد از آن بازه ریست شوند و حجیم نشوند.

## انتشار خودکار با GitHub Actions
workflow زیر اضافه شده است:
- `.github/workflows/build-release.yml`

رفتار:
- روی `workflow_dispatch` قابل اجرا است.
- روی push تگ‌هایی مثل `v1.0.0` اجرا می‌شود.
- خروجی `ConfigCollectorWindows.exe` را در GitHub Release آپلود می‌کند.

### نحوه Release
```bash
git tag v1.0.0
git push origin v1.0.0
```

## نکات فنی
- داده‌ها از صفحات عمومی `t.me/s/...` خوانده می‌شوند.
- برنامه برای سبک بودن از worker thread استفاده می‌کند تا UI هنگ نکند.
- regex استخراج پروتکل‌ها قابل توسعه است.
